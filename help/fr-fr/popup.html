Propriétés du signal

La fenêtre propose deux onglets :

  • Signaux — Données d'infrastructure. Utilisez les flèches si plusieurs signaux sont au même endroit. Près de l'ID Réseau, cliquez sur le logo OSM pour afficher les informations du nœud dans OpenStreetMap, ou sur Localiser sur OpenStreetMap s'il n'est pas encore cartographié.
  • Tags OSM — Affichez, copiez ou ouvrez les tags dans l'éditeur JOSM.

Le bouton Épingler (en haut de la fenêtre) permet de garder un signal en référence.

Mode Diff OSM

Lorsqu'un signal existe déjà dans OpenStreetMap, l'onglet Tags OSM compare les tags générés à ceux intégrés à OSM, avec le code couleur suivant :

  • − railway:* — valeurs présentes uniquement dans OSM, ou en conflit avec la valeur générée.
    Clic droit → Fusionner pour remplacer la valeur générée par celle d'OSM.
  • + railway:* — valeurs générées par l'app absentes d'OSM, ou en conflit avec OSM.
  • ~ railway:* — valeurs fusionnées depuis OSM depuis l'ouverture de la fenêtre.
    Clic droit → Annuler pour restaurer la valeur générée.

Clic droit n'importe où dans la liste pour Tout fusionner et Tout annuler, le cas échéant.

Les tags générés suivent les spécifications OpenRailwayMap — Tagging in France.
Note : Fonctionnalité expérimentale (spécifications en cours d'élaboration).

Certaines valeurs (ex. form, states, speed) sont définies par défaut lorsqu'elles ne sont pas disponibles dans les données ouvertes SNCF. Les tags OSM issus de relevés terrain peuvent être plus précis et peuvent dans ce cas être fusionnés.

Export

Les boutons Copier les tags et Ouvrir dans JOSM exportent les tags cibles actuels (y compris les valeurs fusionnées depuis OSM). Les tags sont triés par ordre alphabétique à l'export ; les clés auxiliaires comme source apparaissent à la fin.
