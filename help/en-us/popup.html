Signal Properties

The popup presents two tabs:

  • Signals — Infrastructure data. Use the arrows to navigate between co-located signals. Beside the Network ID, click the OSM logo for OpenStreetMap node info, or Locate on OpenStreetMap if not yet mapped.
  • OSM Tags — View, copy, or open tags in the JOSM editor.

The Pin button (at the top of the window) keeps a signal for later reference.

OSM Diff Mode

When a signal is already mapped in OpenStreetMap, the OSM Tags tab compares the generated tags against OSM with the following color code:

  • − railway:* — values present only in OSM, or conflicting with the generated value.
    Right-click → Merge to overwrite the generated value with the OSM one.
  • + railway:* — values generated by the app but missing from OSM, or conflicting with OSM.
  • ~ railway:* — values merged from OSM since the popup was opened.
    Right-click → Undo to restore the generated value.

Right-click anywhere on the list for Merge all and Undo all (when applicable).

App-generated tags follow the OpenRailwayMap — Tagging in France specifications.
Note: Experimental feature (specifications are a work in progress).

Some values (e.g. signal form, states, speed) are set to defaults since they are not available in the SNCF open data. OSM tags from field surveys may be more accurate, and can be merged in such cases.

Export

The footer buttons Copy tags and Open in JOSM export the current target tags (including any merged OSM values). Tags are sorted lexicographically on export, so auxiliary keys like source appear at the end.
